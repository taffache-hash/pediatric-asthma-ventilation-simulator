# Pre-submission checklist v0.9

## Software package

- [x] README present and readable.
- [x] LICENSE present.
- [x] NOTICE present.
- [x] CITATION.cff present.
- [x] Requirements file present.
- [x] Scenario YAML files present.
- [x] Engine sanity checks present.
- [x] Streamlit smoke test present.
- [x] Release-integrity test present.
- [x] Figure generation script present.
- [x] Strategy comparison script present.

## Before GitHub public release

- [ ] Check author list and affiliations.
- [ ] Check whether institutional approval or disclaimer is required.
- [ ] Decide final repository name.
- [ ] Add final screenshots or demo GIF if desired.
- [ ] Confirm no private data, patient data, unpublished proprietary material, or hidden credentials are present.
- [ ] Run full test sequence locally.

## Before Zenodo

- [ ] Create GitHub release tag.
- [ ] Confirm title and author metadata.
- [ ] Confirm license metadata.
- [ ] Archive the exact release, not a working branch.
- [ ] Add DOI to README and manuscript after Zenodo generates it.

## Before manuscript submission

- [ ] Verify references manually.
- [ ] Add final DOI/software availability statement.
- [ ] Convert manuscript to journal format.
- [ ] Add conflict-of-interest statement.
- [ ] Add AI/software development disclosure if required by the journal.
- [ ] Add limitations prominently.
